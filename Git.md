# 🛠 Рабочий процесс (Git Flow)

Упрощенный **Feature Branch Workflow**.
Ветка `main` всегда должна содержать рабочий код.

После работы с ветками их необходимо удалять, после каждого PR (Pull Request).

---

## 1. Ветки и именование

* **`main`** — основная ветка. **Прямые коммиты запрещены.** Изменения попадают сюда только через Pull Request (Merge Request).
* **Ветки задач** — создаются от `main` для каждой отдельной задачи.

### Формат имени ветки:
`тип/краткое-название-задачи`

**Типы:**
* `feat/` — новая функциональность (feature)
* `fix/` — исправление ошибки (bug fix)
* `docs/` — изменения в документации
* `refactor/` — правки кода без изменения логики

**Примеры:**
* `feat/auth-login`
* `fix/message-scroll`
* `docs/update-readme`
